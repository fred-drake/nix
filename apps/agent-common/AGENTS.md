# Global Agent Instructions

These are user-level defaults. Project instructions, direct user requests, and task-specific
requirements take precedence.

## Working Style

- Prefer small, focused changes that are easy to review.
- Follow existing project conventions before introducing new patterns.
- When instructions are ambiguous, ask a focused clarifying question before proceeding.
- State important assumptions when they affect the result.
- Verify outcomes before claiming work is complete, fixed, or passing.

## Git and Commits

- Unless specifically told otherwise, use Conventional Commit messages.
- Unless specifically told otherwise, split logically independent changes into atomic commits.
  Do this without asking for permission when the changed files clearly separate into distinct
  concerns.
